# 応答生成〜音声合成，音声出力の遅延について
LLMでの応答生成に時間がかかるため，応答を返すタイミングが遅くなってしまう．

ユーザ発話終了 → ASR開始・終了 → LLM応答生成（2〜3秒） → 音声合成 → 音声再生

音声対話は，一般的には，相手の発話末から1.5秒間が空くと対話が破綻する．なのでこれを避けたい．
最近の手法としては，
- 発話終了（話者交替点）を検出したら，とりあえず短い相槌的発話をTTSして再生する（これは短いし瞬間的に終わる）
- それをしている間に，ASR結果をLLMに入れて応答生成・音声合成を行う．

LLMも，重たすぎるものであれば，この処理でも間に合わなくて遅延してしまうが，軽いLLMであれば，スムーズに対話ができる．
